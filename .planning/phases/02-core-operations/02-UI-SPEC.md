# Phase 2 — UI Design Contract (Terminal CLI)

> Visual and interaction contract for a Python terminal CLI app.
> No web frontend. "UI" = terminal text layout, prompt patterns, output formatting.
> Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — Python stdlib input() + print() only |
| Icon library | Unicode box-drawing chars + checkmark (✓) — no external dependency |
| Font | Terminal monospace (user's system font) |

No shadcn gate applies. This is a pure Python terminal script with zero external dependencies.

---

## Spacing Scale

Terminal spacing is measured in blank lines and leading spaces, not pixels.

| Token | Value | Usage |
|-------|-------|-------|
| indent | 2 spaces | Menu option indentation: `"  1) ..."` |
| separator | 1 blank line | Between menu header and options; between prompt and response |
| section-break | blank `print()` | Before every menu reprint (existing pattern in show_menu()) |
| inline-gap | ` \| ` (space-pipe-space) | Summary field separator in one-liner result |

Exceptions: none. All existing Phase 1 patterns reused.

---

## Typography

Terminal text has no font sizes or weights. The equivalent contract is line structure.

| Role | Pattern | Example |
|------|---------|---------|
| Header | `=== {title} ===` | `=== Organizador \| E:\\ MEDIOS ===` |
| Menu option | `  {N}) {label}` (2-space indent) | `  1) Organizar videos y juegos` |
| Toggle option | `  5) Dry-run: {STATE}` (STATE = ON or OFF, uppercase) | `  5) Dry-run: OFF` |
| Prompt | `{Label}: ` (no newline, trailing space) | `Opcion: ` |
| Result one-liner | `✓ Procesados: N \| Movidos: N \| Saltados: N \| Errores: N` | see Copywriting section |
| Error inline | plain sentence, no prefix symbol | `Opcion invalida.` |
| Dry-run line | `[DRY-RUN] {description}` | `[DRY-RUN] MOVER: archivo.mkv -> Series\Show\T1\` |

---

## Color

Terminal output uses no color (Windows cmd default). No ANSI escape codes. Windows
compatibility and zero-dependency constraint prohibit colorama or similar.

| Role | Terminal Equivalent | Usage |
|------|---------------------|-------|
| Dominant surface | default terminal background | All output |
| Secondary | N/A | Not applicable |
| Accent | `✓` prefix on success one-liner | Result summaries only |
| Destructive | Uppercase label in prompt | Confirmation prompts for irreversible actions |

Accent (`✓`) reserved for: the final summary one-liner after each completed operation only.

## Menu Layout Contract

### Main Menu (show_menu)

The full menu as it must render after Phase 2 is wired (extends existing Phase 1 stub):

```
                                   ← blank line (section-break)
=== Organizador | E:\ MEDIOS ===
  1) Organizar videos y juegos
  2) Aplicar rename_plan.tsv
  3) Revertir ultima operacion      ← stub, Phase 3
  4) Detectar incoherencias         ← stub, Phase 4
  5) Dry-run: OFF                   ← toggles to "Dry-run: ON"; no blank line between 4 and 5
  0) Salir
Opcion: _
```

Rules:
- Option 5 label text changes with executor.dry_run state: `OFF` when False, `ON` when True.
- No blank lines between individual options.
- One blank line before the header line (existing `print()` pattern).
- Prompt `Opcion: ` has no newline — `input("Opcion: ")`.

### Dry-run State Indicator

When dry_run is ON, the menu itself communicates the state. No banner, no warning, no
extra line. The user sees `5) Dry-run: ON` and that is sufficient (D-02: no confirmation
needed before running operations).

## Operation Output Contract

### During execution (per-file feedback)

Each move or rename that would happen (or does happen) produces one log line via `logger.info()`.
These go to the rotating file log and the console handler simultaneously.

| Scenario | Output line format |
|----------|--------------------|
| Real move | `INFO MOVE: {src} -> {dst}` |
| Dry-run preview | `INFO DRY-RUN MOVE: {src} -> {dst}` |
| Hard-blocked (ROM ext) | `WARNING SKIP (no-touch ext): {src}` |
| Protected path skip | `WARNING SKIP (protected path): {src}` |
| Permission error | `ERROR DENY: {src} (acceso denegado)` |
| OS error | `ERROR ERR: {src} : {exc}` |

Phase 2 operations MUST NOT use `print()` for per-file output. All per-file feedback
routes through `logger` (existing pattern — keeps console clean, full detail in file log).

### After execution — result summary (MENU-03)

One `print()` call immediately before returning to the menu loop:

```
✓ Procesados: 47 | Movidos: 32 | Saltados: 12 | Errores: 3
```

Rules:
- Always exactly one line. Never multi-line.
- `✓` prefix always present regardless of error count.
- Field order fixed: Procesados → Movidos → Saltados → Errores.
- Field separator: ` | ` (space, pipe, space).
- No trailing newline beyond what print() adds.
- Printed to stdout (print(), not logger). This is user-visible feedback, not a log entry.
- Source: CONTEXT.md Claude's Discretion — MENU-03 format decision.

## Copywriting Contract

All copy in Spanish (project language). No accents on uppercase (terminal compatibility).

| Element | Copy |
|---------|------|
| Primary CTA — organize | `Organizar videos y juegos` |
| Primary CTA — rename | `Aplicar rename_plan.tsv` |
| Dry-run toggle OFF→ON | `5) Dry-run: OFF` (user selects → switches to ON) |
| Dry-run toggle ON→OFF | `5) Dry-run: ON` (user selects → switches to OFF) |
| Result success line | `✓ Procesados: {N} \| Movidos: {N} \| Saltados: {N} \| Errores: {N}` |
| TSV not found | `No se encontro rename_plan.tsv en {root}. Coloca el archivo y vuelve a intentarlo.` |
| TSV empty | `El archivo rename_plan.tsv esta vacio. Agrega entradas y vuelve a intentarlo.` |
| TSV row missing column | `Fila {N} ignorada: faltan columnas old_path / new_path.` (logger.warning, not print) |
| No video files found | `No se encontraron archivos de video para organizar.` (print before summary line) |
| Invalid menu input | `Opcion invalida.` (existing Phase 1 pattern — do not change) |
| PC/Steam folder skipped | Silent skip — no output, no warning (D-04: by design) |
| Empty folder removed | Silent removal — logged to file only (logger.debug), not printed |
| Destructive action (none in Phase 2) | N/A — Phase 2 has no destructive operations requiring confirmation |

No empty-folder cleanup confirmation prompt. `os.rmdir` only removes already-empty folders
(non-destructive by definition). No confirmation needed.

## Interaction Flow Contract

### Option 1 — Organizar videos y juegos

```
Opcion: 1
                              ← operation runs silently (logger lines to file+console)
✓ Procesados: 47 | Movidos: 32 | Saltados: 12 | Errores: 3
                              ← menu reprints immediately
=== Organizador | E:\ MEDIOS ===
...
```

With dry-run ON, identical flow. No extra prompt. No "(Dry-run)" suffix on summary line.
The menu's `5) Dry-run: ON` already communicates the mode.

### Option 2 — Aplicar rename_plan.tsv

```
Opcion: 2
                              ← TSV read; if not found: error message then menu
✓ Procesados: 12 | Movidos: 12 | Saltados: 0 | Errores: 0
                              ← menu reprints
```

TSV error path:
```
Opcion: 2
No se encontro rename_plan.tsv en E:\. Coloca el archivo y vuelve a intentarlo.
                              ← menu reprints immediately (no summary line on error abort)
```

### Option 5 — Toggle dry-run

```
Opcion: 5
                              ← no output, no confirmation
=== Organizador | E:\ MEDIOS ===
  ...
  5) Dry-run: ON              ← state reflected immediately in next menu reprint
  0) Salir
Opcion: _
```

Toggle is instantaneous: `executor.dry_run = not executor.dry_run`, then loop continues.

## Scan Exclusion Contract (Discretion area from CONTEXT.md)

When scanning the drive root for files to organize (ORG-01, ORG-02), these top-level
folders are excluded from the traversal:

| Folder | Reason |
|--------|--------|
| `_organizer_logs\` | Script metadata — never touch |
| `Series\` | Already organized destination — skip to prevent double-move |
| `Peliculas\` | Already organized destination — skip to prevent double-move |
| `Juegos\` | Already organized destination — skip to prevent double-move |
| All paths in `SKIP_PATH_PARTS` | Phase 1 hard block — already enforced by Executor |

For ORG-03 (game console folders), additionally exclude: `PC\` and `Steam\` (D-04).

## TSV Path Format (Discretion area from CONTEXT.md)

`rename_plan.tsv` columns `old_path` / `new_path` contain absolute paths (e.g.
`E:\Videos\old name.mkv`). Consistent with Renombrar.ps1 ground truth and with
how `Executor.move(src, dst)` expects `Path` objects constructed from absolute strings.
